看样子是一个搞錯了 RAM 和 Storage 的大学生提出的問題，正好最近与根哥聊天時很有吐槽意思，那不妨來看看這些「計算机術語」

查閱常见手机官网可知：大部分国內厂商習慣將 RAM 稱爲运行內存，也就是題主所理解的內存，將 Storage 稱爲机身存儲。值得注意的是 Apple 并未提供任何与 RAM 有關的信息[^1]。將目光投向 PC 厂商呢？基本是內存＋硬盤的組合。

那么哪种名稱比較好呢？先來看看英語原名。在馮諾依曼体系下，运算器、控制器、存儲器、输入设备和输出设备共同构成了現代計算机，單看存儲器又有：

* 存儲器 (Memory)
   * **主存**儲器 (Main **Memory**)
      * RAM（随机存取存儲器）(Random Access Memory)
      * ROM（只讀存儲器）(Read-Only Memory)
   * 輔助**存儲**器 (Secondary **Storage**)
      * 磁存儲器 (Magnetic Storage)
      * 光存儲器 (Optical Storage)
      * 固態存儲器 (Solid-State Storage)
         * 固態硬盤（SSD）(Solid-State Drive)
         * U 盤 (USB Flash Drive)
         * 存儲卡 (Memory Card)
   * 高速**緩**冲**存**儲器（Cache）
   * 寄存器 (Registers)
      * 通用寄存器 (General-Purpose Registers)
      * 專用寄存器 (Special-Purpose Registers)

其实內存 Memory 相對應的是外存 Storage ，即外部存儲，它不参与[^2]运算流程，所以叫外部。一切程序和數据都是先進內存，然后 CPU 來計算，CPU 只跟內存打交道，不跟硬盤打交道：即外部存不属于核心部件的一部分。所以內存有且僅有一个，跟 CPU 打交道的那个存儲數据和程序的东西就叫內存。

那么在我這里「运行內存」就是一个很爛的翻譯，大可不必留着前两个字，和电腦统一就是很好的叫法。

現在回到問題，你需要多大「內存」的手机？

我的设备是 6 GB 的內存， 128 GB 的存儲，這對我來說是足够使用的。如果單獨討論存儲，**武斷的認爲某个存儲容量够用或不够用都是扯淡**。如果我有男朋友的話我一定不会給他買 XS 的女装：在不知道對方实际情况下，你是如何斷定對方适合或不适合穿這件 XS 号 [衣服](https://world.taobao.com/item/640826102183.htm?spm=a21wu.10013406-tw.taglist-content.5.53ed1db1z6NdC5)的？手机存儲規格建議也是同理，弄清楚自己的需求，喜不喜歡打游戲、聊天、拍照，都会影响你最終的選擇。不然哪怕你買一个 1 TB 的手机，多拍几端 4 K 120 HZ 视頻又要開始內存焦慮了。那么討論一下一般性的情况呢？

<br>

# 大就是好😋

收集而不消化完全沒意义，如果消化不掉也沒有存留的意义，消化掉了內化爲自己的知識也沒必要留存本体，总有新东西出現的。

<br>
也許每个人都有這个文件不能删的理由，但如果你重装系统忘备份就会明白其实沒有多少是你真正留恋的，畢竟只需要知道名字就可以从网上找回來，有意思的是很多人就是不知道名字是什么。

那么照片呢 ？ ~~我所記錄的成长軌迹多么值得回味啊~~ 有多少人是只照不整理的，或者精心拍攝只是留在朋友圈。現在每个人可能都有几千張照片，也懶得冲洗出來，只是在电腦上随便看看。但你越覺得一件事稀松平常，你就越不会放入你大腦的內存，太廉价而且可以不斷獲得新的紀錄。在整理照片上需要的只是一个不删的理由而不是删除的理由，把剩下的照片都冲洗出來，這样才真正有記錄的价值。不要总想着有一天我会用到，到了那一天你根本就找不到。

处理完這些大体積的資料再除去游戲之外的空间，你還需要多大的存儲呢？

吃完飯回來发現其实大部分人留下装抖音的空间就够了，短视頻都要两倍速看的人，不需要很多需要記錄的空间。



[^1]:看起來大部分公司并不希望消費者知道 Cache 和 Registers 的數据，某些公司甚至不希望用户知道 Meomory (這里指的是主存，也就是內存)的數据，但无一例外地都標明了自己商品的 Storage （存儲）。
[^2]:這点存疑，雖然書上這么寫，但是我的 C 盤真的部分叫做 virtual memory
